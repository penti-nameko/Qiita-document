# 0.初めに
helmchartを使うと何故かoutlineを最新版にすることができなかったのでいっそdockerコンテナでデプロイすればええやんと思ったのでやり方の解説
# 1.ingress.yamlの例
```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: outline-ingress
  namespace: outline
  annotations:
    kubernetes.io/ingress.class: nginx
    nginx.ingress.kubernetes.io/proxy-body-size: "100m"
spec:
  rules:
    - host: wiki.reeldev.jp
      http:
        paths:
          - path: /
            pathType: ImplementationSpecific
            backend:
              service:
                name: outline-service
                port:
                  number: 3000
```

# 3.deployment.yamlの例
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: outline
  namespace: outline
spec:
  replicas: 1
  selector:
    matchLabels:
      app: outline
  template:
    metadata:
      labels:
        app: outline
    spec:
      containers:
        - name: outline
          image: outlinewiki/outline:latest

          ports:
            - containerPort: 3000
          volumeMounts:
            - name: outline-storage
              mountPath: /var/lib/outline/data
          env:
            - name: NODE_ENV
              value: "production"
            - name: PGSSLMODE
              value: "disable"
            - name: URL
              value: "https://wiki.reeldev.jp"
            - name: REDIS_URL
              value: "redis://:redispassword@redis-service:6379"
            - name: AUTH_URI
              valueFrom: { secretKeyRef: { name: outline-secrets, key: authUri } }
            - name: TOKEN_URI
              valueFrom: { secretKeyRef: { name: outline-secrets, key: tokenUri } }
            - name: USERINFO_URI
              valueFrom: { secretKeyRef: { name: outline-secrets, key: userInfoUri } }
            - name: SECRET_KEY
              valueFrom: { secretKeyRef: { name: outline-secrets, key: SECRET_KEY } }
            - name: "displayName"
              valueFrom: { secretKeyRef: { name: outline-secrets, key: displayName } }
            - name: UTILS_SECRET
              valueFrom: { secretKeyRef: { name: outline-secrets, key: UTILS_SECRET } }
            - name: DATABASE_URL
              valueFrom: { secretKeyRef: { name: outline-secrets, key: DATABASE_URL } }
            # Discord
            - name: DISCORD_CLIENT_ID
              valueFrom: { secretKeyRef: { name: outline-secrets, key: DISCORD_CLIENT_ID } }
            - name: DISCORD_CLIENT_SECRET
              valueFrom: { secretKeyRef: { name: outline-secrets, key: DISCORD_CLIENT_SECRET } }
            # GitHub
            - name: GITHUB_CLIENT_ID
              valueFrom: { secretKeyRef: { name: outline-secrets, key: GITHUB_CLIENT_ID } }
            - name: GITHUB_CLIENT_SECRET
              valueFrom: { secretKeyRef: { name: outline-secrets, key: GITHUB_CLIENT_SECRET } }
            - name: GITHUB_APP_ID
              valueFrom: { secretKeyRef: { name: outline-secrets, key: GITHUB_APP_ID } }
            - name: GITHUB_APP_PRIVATE_KEY
              valueFrom: { secretKeyRef: { name: outline-secrets, key: GITHUB_APP_PRIVATE_KEY } }
            # OIDC
            - name: OIDC_CLIENT_ID
              valueFrom: { secretKeyRef: { name: outline-secrets, key: OIDC_CLIENT_ID } }
            - name: OIDC_CLIENT_SECRET
              valueFrom: { secretKeyRef: { name: outline-secrets, key: OIDC_CLIENT_SECRET } }
            - name: OIDC_AUTH_URI
              value: "https://auth.uniproject.jp/authorization"
            - name: OIDC_TOKEN_URI
              value: "https://auth.uniproject.jp/token"
            - name: OIDC_USERINFO_URI
              value: "https://auth.uniproject.jp/userinfo"
            - name: OIDC_DISPLAY_NAME
              value: "UniQue"
            - name: OIDC_SCOPES
              value: "openid profile email"
            - name: FILE_STORAGE
              value: "local"
            - name: STORAGE_LOCAL_ROOT_DIR
              value: "/var/lib/outline/data"
      volumes:
        - name: outline-storage
          persistentVolumeClaim:
            claimName: outline-storage-pvc
```
secret.yamlは自分で生成してください(セキュリティの観点から)
# 4.pvc.yamlの内容
```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: outline-storage-pvc
  namespace: outline
spec:
  accessModes: ["ReadWriteMany"]
  storageClassName: "nfs-client"
  resources:
    requests:
      storage: 10Gi
---
# PostgreSQL用
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: postgres-data-pvc
  namespace: outline
spec:
  accessModes: ["ReadWriteOnce"]
  storageClassName: "nfs-client"
  resources:
    requests:
      storage: 8Gi
---
# Redis用

  name: redis-data-pvc
  namespace: outline

  accessModes: ["ReadWriteOnce"]

  resources:

```

# 6.設定手順
1.seacretを適切に調整して設定します。
2.githubなどにpushし、Argocdでデプロイします。
3.その後、ingress等を調整し、動作確認します。
問題がなければこれで済むはずです。
