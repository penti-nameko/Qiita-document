# 0.初めに
この記事Linuxのエマージェンシーモードに入ってk8sが死んだときの対処で参考になるかもしれない記事として書く。
# 1.修復
まずは、エマージェンシーモードなのかどうかを切り分けするために以下を実行
```
dh -h
```
```
mount | grep "on / "
```

```
sudo journalctl -u containerd.service -n 20 --no-pager
```

```

touch ~/test_file && rm ~/test_file
sudo systemctl restart kubelet
sudo systemctl status containerd
```

こんな感じで動作するようになれば確定です。
ディスクチェックを行わせて修復しましょう
```
sudo reboot
```
で再起動後、
```
sudo fsck -f -y /dev/mapper/ubuntu--vg-ubuntu--lv
sudo systemctl restart kubelet
```
これでエラー無く完了すれば作業終了です。
