# 0.初めに
筆者はウェブブラウザゲームをよくやるが、その際、
ハードウェアアクセラレーションによって意図しない動作をする場合がある。
今記事ではそんなハードウェアアクセラレーションについて開発者目線で語る。
# 1.ハードウェアアクセラレーションとは
まず、大前提としてCPUは1フレームごとにピクセル単位で処理をするため、
GPUと比べて処理は必然的にゆっくりである。
それを、GPU内に確保されたレイヤーに取り込んでGPUで処理をすることで高速化する。
その際、当然ではあるがデータはメモリにキャッシュし、読み込みを高速化しようとする。
しかし、これが問題を起こしやすい。次項で解説する。
# 2.当機能の有効化で起こりやすい問題とは
前項で解説したように、この機能はメモリを食いやすい傾向がある。
これがきっかけとなって、必要な処理を実行できずクラッシュすることがある
もとより、この機能はCPUとGPUとのデータ通信が増えるので場合によっては逆効果になることすらある
その上、アクセラレータの命令仕様や精度でブレる場合があることである。
# 3.では、利点は？
このシステムはシンプルに言えばCPUの負荷が大きい処理をGPUに肩代わりさせる仕組みなので、
負荷が重くなりやすい場合はシステムの高速化を狙える。
逆に言えばCPUをあまり使わない処理には効果が薄いのだが。
とはいえ、メモリは多くともプロセッサに難ありなPCであればシステムの速度向上なども狙える点が大きい。
# 4.開発者として考えなければならないこと
やはり注意すべきは前述のとおりレイヤーのスキップなどによるクラッシュである。
なので、アニメーションなどの動きが完了するまで次の処理を開始させないなどの工夫が必要である。
