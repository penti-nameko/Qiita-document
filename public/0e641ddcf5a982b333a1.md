# 2.そもそもk8sとは

# 3. マシンの種類と活用
マシン、と一言でいっても当然種類があります。
物理マシン、仮想マシン.....その中にも種類が....というとわかりにくいので
ざっくり分けると

物理マシン：我々の言う物理サーバーやPC
仮想マシン：ホストとなる物理マシンの上で運用される仮想的なコンピューター

となります。
要は何が言いたいかというと、
### 実態があるかないかです
k8sクラスタを作る上ではあまりどちらでも問題はないですが仮想マシンのほうが拡張性はあります。
当然、ニーズに応じて増やしたり減らしたりしやすいからですが。
とはいえ、k8sはエラーで止まったコンテナ等を容赦なく作り直すためデータ保持を設定する必要があります。
であれば、バックアップなどもそうですが実機(物理マシン)を増やしたほうが安全性は高まるでしょう。
# 4.仮想化の種類
じゃあ実際に仮想化するとなるとどういうシステム形態になるのかというと
![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/4434251/e27a0a53-9aa1-4d1d-86fb-1d8c5c6b4e57.png)
方式がこの三種類になります。
左がホスト型といい、
Oracle VM VirtualBox等のWindows,Mac,Linux等で幅広く使いやすく、扱いやすい形になります。
中央はハイパーバイザーというもので、ホストOSを使用しない分リソースを削減できます。
主な例はMicrosoft Hyper-Vなんかがそれにあたります。
そして残ったコンテナ型は、共通のリソースで足りないものを追加して、"アプリケーションに必要な環境"を構築するため、リリース期間の短縮などが可能な方式です。
主な例として、DockerやKubernetesがまさにそれです。


では、そんなk8s等をどのように運用していくのか.....というのはメインシステム編に続きます
